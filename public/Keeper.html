Keeper Website

Live Website: keeper-phi-five.vercel.app

Keeper is a full-stack web application designed to securely manage and organize user-created notes. The application was built using React for the frontend and Node.js with Express for the backend, following a modern single-page application (SPA) and client–server architecture.

The application includes the following key features:

  • User Authentication: Supports secure local authentication using email and password. Google Sign-In via OAuth 2.0 is implemented and fully functional in the development environment.
  • OAuth Integration Status: Google OAuth 2.0 is currently unavailable in the production environment because the application has not yet completed Google’s OAuth verification process. Local authentication remains fully functional.
  • Authorization & Session Management: Uses JWT-based authentication stored in HTTP-only cookies to securely manage user sessions and protect API routes.
  • Note Management: Authenticated users can create, view, update, and delete personal notes, with all data scoped to the owning user.
  • RESTful API: The backend exposes RESTful endpoints for authentication and CRUD operations, maintaining clear separation between client and server.
  • Security & Configuration: Implements CORS configuration, environment variables, and server-side token validation to ensure secure cross-origin communication.
  • Deployment & Hosting: The frontend and backend are deployed as separate services using free-tier hosting platforms. As a result, initial requests may experience increased response time due to server cold starts. This application was built primarily for learning and practice purposes rather than production-scale performance.

This project was developed to practice modern full-stack development, authentication workflows, and real-world deployment considerations.
